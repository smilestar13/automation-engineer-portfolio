# Python Automation

## Overview

This directory contains a collection of production-inspired Python automation examples.

The projects demonstrate different approaches to automation commonly used by Automation Engineers, RPA Developers, and DevOps Engineers in enterprise environments.

Rather than focusing on a single business domain, these examples showcase reusable automation techniques that can be adapted to many real-world scenarios.

---

## Included Projects

### API Metrics Monitor

Demonstrates how Python can periodically collect data from REST APIs, process responses, and maintain historical metrics for reporting and monitoring purposes.

**Technologies**

- Python
- REST API
- Requests
- CSV Processing

---

### Playwright Browser Automation

Demonstrates browser automation using Playwright, including browser profile management, browser extension integration, JavaScript execution, dynamic UI interaction, and continuous dashboard monitoring.

**Technologies**

- Python
- Playwright
- Chromium
- JavaScript

---

### Selenium Email OTP Login

Demonstrates automated email verification using Selenium and IMAP.

The workflow authenticates to a protected web portal, retrieves a one-time verification code (OTP) from Gmail via IMAP, and completes the login process automatically.

This type of automation is commonly used as a reusable authentication component executed by Bash scripts, scheduled tasks, health checks, or larger RPA workflows.

**Technologies**

- Python
- Selenium
- IMAP
- Gmail
- Threading

---

### Text to CSV Parser

Demonstrates automated processing of semi-structured text files and conversion into structured CSV datasets suitable for reporting, migration, or further processing.

**Technologies**

- Python
- Regular Expressions
- CSV
- File Processing

## Repository Purpose

The goal of this collection is to demonstrate practical Python automation skills rather than provide complete production applications.

Each project focuses on solving a specific automation problem while following clean coding practices and maintainable project organization.

---

## Disclaimer

Some repositories contain **sanitized demonstrations** of production workflows.

To protect confidential information, business-specific implementation details such as URLs, selectors, credentials, API endpoints, datasets, and workflow logic have been replaced with generic placeholders.

The architecture, programming patterns, and automation techniques have been preserved for demonstration purposes only.

Any similarity to real systems, services, or data is purely coincidental.
